# DELIVERABLES

Phân tích toàn bộ code hiện có trước khi sửa.

Chỉ nâng cấp những phần cần thiết, không phá vỡ phần đang hoạt động tốt.

Nếu cần tạo file mới thì phải đầy đủ và có thể chạy được ngay.

Không dùng pseudo code.

Không dùng "...".

Không viết "implement here".

Không bỏ trống component.

Không thay đổi API hiện có nếu không thật sự cần thiết.

Nếu phải thay đổi API thì phải đảm bảo tương thích ngược hoặc nêu rõ lý do.

Sau khi hoàn thành, tiếp tục tự kiểm tra và cải tiến:

* Tối ưu UI/UX.
* Tối ưu hiệu năng.
* Kiểm tra responsive.
* Kiểm tra accessibility.
* Loại bỏ code dư thừa.
* Giảm bundle size.
* Chuẩn hóa cấu trúc dự án.
* Sửa mọi lỗi TypeScript và ESLint.
* Đảm bảo dự án có thể chạy ngay bằng `npm install` và `ng serve`.

Không dừng lại sau phiên bản đầu tiên. Hãy tiếp tục lặp lại chu trình "phân tích → cải tiến → tối ưu" cho đến khi chất lượng đạt mức production-ready.
